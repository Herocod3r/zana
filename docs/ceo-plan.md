# CEO Plan: Zana — Minimal Agent Orchestrator
Generated by /plan-ceo-review on 2026-04-06
Branch: main | Mode: SELECTIVE EXPANSION
Repo: Zana

## Vision

A desktop app that organizes coding agents by project and worktree-isolated workspace. Open it, pick a project, spawn agents in parallel terminal panes, watch them work, quit, resume exactly where you left off. The whole point is being noticeably better than alt-tabbing between terminal windows, without the bloat that makes Superset feel slow.

The thin core ships first. Extensions come later, only after dogfooding reveals what they actually need to do.

## Accepted Scope (added to A+ baseline)

- **Terminal activity indicator dot** — colored dot per terminal pane (green = recent output, yellow = recent-ish, gray = idle). Driven by last-byte-time per PTY in the Go terminal manager. 1Hz status updates pushed to frontend. Stays agent-agnostic.

## Approach: A+ (Minimal Core with Extension Seams)

Same as office-hours Approach A, plus 2-3 internal interfaces in Go and Vue component slots where extensions would plug in later. No extension loader, no plugin system. Just clean seams: `TerminalProvider`, `WorkspaceAction`, `SidebarPanel` (or similar). The point is to avoid a future refactor when extensions come.

## Deferred to TODOS.md

- **Workspace notes panel** — markdown text per workspace, "what is this agent supposed to be doing again?"
- **Cmd+P fuzzy switcher** — power-user navigation across projects/workspaces/terminals
- **Per-project `.zana/init.sh`** — setup script run on workspace creation

## Cross-Model Tensions Resolved (Codex outside voice)

| # | Codex challenge | Decision | Why |
|---|-----------------|----------|-----|
| 1 | "Resume" promise is dishonest (v1 only restores layout + cwd) | **1B: Persist scrollback** | Save last 256 KB of rendered output per pane. Replay on reopen. User runs `claude resume` for the agent's logical state. Zana preserves visual context only. |
| 2 | Multi-pane is bloat, just do one terminal per workspace | **2C: Multi-pane stays** | Mental model is hierarchical (projects → workspaces → panes), not multiplexing. Panes are first-class. |
| 3 | Git UX is the product, not a "shell out" footnote | **3A: Rich New Workspace dialog** | Workspace name (auto-generated, swappable), branch name, base branch, new/existing toggle, submodule warning. |

**Adjusted framing:**
- Cross-platform → "macOS-first in practice." Linux is a fast follow. Windows is post-v1.
- Distribution → keep Homebrew tap, but acknowledge it's a one-config-file investment, not the focus.
- Extension seams → kept (3 interfaces total), but documented as load-bearing for v1, not extension API design.
